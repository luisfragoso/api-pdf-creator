# Security Policy

## Supported Versions

This project is provided as-is. Security fixes may be released on a best-effort basis.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it **privately**.

- Do **not** open a public GitHub issue for security reports.
- Email: **security@REPLACE_ME**

Include as much detail as possible:

- Steps to reproduce
- Affected endpoint(s)
- Proof of concept (if available)
- Impact assessment

## Disclosure Policy

We will acknowledge receipt of your report and make a best-effort attempt to provide updates.

When a fix is available, we will coordinate a responsible disclosure timeline when possible.
